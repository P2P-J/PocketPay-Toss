# 작은 모임 (pocketpay) — 앱인토스 수익화 전략 메모

> 결론: 인앱광고·"모임 개수 과금"은 빼고, **모임 생성은 무료로 풀어 성장 → 활발한 방장 대상 기능형 Pro(IAP)**. 그리고 **챌린지(7/1~26 AU 심사) 끝나고** 도입.

## 1. 앱인토스 수익화 옵션 (우리 기준)
| 방식 | 용도 | 우리 적합도 |
|---|---|---|
| 인앱광고 | 노출형 수익 | ❌ UX 해침, 비게임 광고 규정 까다로움 |
| **인앱결제(IAP)** | 디지털 "프리미엄 기능 해제·권한·콘텐츠"(비소모성/구독) | ✅ 정답 |
| 토스페이 | **실물 상품·서비스** 결제 | ❌ 디지털 기능엔 불가 |

- IAP 공식 예시 = "프리미엄 기능 해제, 광고 제거, 특정 콘텐츠 이용 권한" → 우리 모델과 동일.
- IAP는 **일회성(비소모성)** + **구독** 둘 다 지원.

## 2. 사용자 아이디어("모임 2개부터 결제") 평가 — 비추천
1. 대부분 유저는 모임 1개 → 결제 전환 거의 없음(작은 TAM).
2. **모임 생성 = 성장 엔진**(생성→초대 QR/코드→신규 유입). 여기 벽 세우면 바이럴/AU를 스스로 차단. 챌린지 1차 심사가 AU라 치명적.
3. 돈 낼 사람이 가장 활발한 방장인데, 격려 대신 과금으로 누르면 역효과.
→ **개수 제한보다 "기능 깊이"로 과금**이 정석.

## 3. 추천 모델: 기능 기반 Pro (방장/모임 단위 과금)
모임 생성·기본 기록은 **무료 유지**. 활발한 방장이 원하는 고급 기능을 Pro로:
- 거래 내역 **무제한 기간**(무료=최근 3개월) + **엑셀/CSV 내보내기**
- **영수증 OCR 무제한**(무료=월 N장 — 핵심이자 서버 비용 발생점)
- 고급 분석·예산 알림, 멤버별 리포트
- 멤버 수 확대(무료 N명까지)
- 정산 리포트 고급 공유(PDF/이미지)

**과금 형태**
- A. 월/연 구독 — 정석이지만 캐주얼 모임엔 거부감.
- B. 일회성 "Pro 평생"(비소모성) — 진입장벽 낮음.
- **추천: 초기 B → 사용자 쌓이면 A 도입.** 과금 주체 = 모임을 만든 **방장**(모임 전체가 혜택).

## 4. 정책·타이밍 주의
1. **챌린지 기간(7/1~26)엔 유료화 금지.** AU 최대화 후, 출시 뒤 별도 업데이트로 수익화.
2. **다크패턴 금지**(앱인토스 디자인 정책): 가격 명확·닫기 쉽게·작업 중간 강제 차단 X. "모임 만들다 결제창" 같은 건 반려 위험.
3. IAP 비소모성/구독 → **기기변경 시 복원(restore)** 필수.
4. **공유 백엔드 주의**: 토스 유저는 기존 App Store 작은모임과 별도 계정. Pro 권한 필드를 공유 User/Team에 추가 시 메인앱과 안 겹치게 설계.
