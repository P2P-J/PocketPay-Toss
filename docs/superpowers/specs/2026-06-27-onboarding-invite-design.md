# 온보딩 & 초대 플로우 설계 (작은 모임)

작성일: 2026-06-27 · 상태: 설계 승인 대기

## 1. 배경 / 문제

앱인토스(토스 로그인)로 신규 가입한 유저는 백엔드가 `name`/`nickname`만 만들고
**`handle`(고유 ID)을 만들지 않는다.** 그런데 멤버 초대는 handle 기반이라,
**핸들 없는 신규유저는 초대를 주고받을 수 없다.** 또한 홈 빈 상태는 텍스트 한 줄뿐이라
**첫 모임을 만들거나 참가할 진입로 자체가 없다.** 초대 토큰 백엔드(`invite-token`/`join/:token`)는
존재하나 프론트가 미연결이고, 멤버가 모임을 나가는 기능도 없다.

## 2. 목표 (성공 기준)

- 신규유저가 [환영/로그인 → 프로필(이름·닉네임·핸들) → 팀 선택(만들기/참가)] 흐름을 끊김 없이 통과한다.
- 방장이 초대 코드 + QR을 만들어 단톡방에 공유할 수 있다.
- 누구나 초대 코드(텍스트)로 모임에 참가할 수 있다. QR은 표시 + (딥링크) 외부 카메라 스캔.
- 멤버가 모임을 나갈 수 있다.
- 거래 추가에서 영수증 첨부가 맨 위에 있다.
- 모든 핵심 엣지케이스(핸들 중복/쿨다운, 코드 만료/이미 멤버, 미로그인 딥링크)가 처리된다.

## 3. 백엔드 (이미 존재 — read-only repo)

- 신규 판별: 토스 신규유저는 `user.handle`이 비어있음.
- `POST /teams/:teamId/invite-token` → `{ token, expiry }` (유효 토큰 있으면 재사용)
- `POST /teams/join/:token` → `{ alreadyMember, team }` (만료/무효 시 에러)
- `DELETE /teams/:teamId/members/me` (leaveTeam — 방장은 위임 후 가능)
- `PATCH /account/handle` (updateHandle) — **30일 1회** 변경 쿨다운(`handleChangedAt`), 신규는 쿨다운 없음
- `GET /account/handle/check` (checkHandle) — 중복 확인
- 핸들 규칙: `/^[a-z0-9_]{3,20}$/` (소문자 영문·숫자·`_`, `-` 불가)

## 4. 진입 게이트 (실모드)

```
앱 실행 → checkAuth (+ 로그인 후 refreshUser로 user.handle 확보)
 ├ 토큰 없음              → /login (환영)
 ├ 로그인 + handle 없음   → /onboarding (프로필) ← 신규
 ├ 로그인 + 팀 0개        → 홈 = 팀 선택 빈 상태
 └ 로그인 + 팀 있음       → 홈
```
- 프리뷰(PREVIEW_MODE): 게이트 건너뛰고 더미 홈. 온보딩/초대도 로컬 더미로 미리보기 가능.
- 미로그인/미온보딩 상태로 들어온 초대 딥링크 토큰은 storage에 stash → 로그인·온보딩 완료 후 자동 소비.

## 5. 화면 / 컴포넌트

### 5.1 환영·로그인 (`/login` 리디자인)
스파클 로고 + "작은 모임" + 한 줄 소개 + 브랜드 그라데이션 배경 + **[토스로 시작하기]**.
별도 소개 슬라이드 없이 단일 화면이 소개+로그인을 겸한다. 로그인 실패 시 에러 표시.

### 5.2 온보딩 프로필 (`/onboarding`, 신규 전용·뒤로가기 차단)
- 필드: 실명 / 닉네임 / **핸들** — 각 필드에 의미 설명(핸들 = "친구가 초대할 때 쓰는 고유 ID").
- 핸들: 형식검증(소문자·숫자·`_` 3~20자) + 실시간 중복확인(checkHandle, 디바운스) + *"30일에 한 번만 바꿀 수 있어요. 신중히 정해주세요."*
- 완료 → `updateProfile`(name/nickname) + `updateHandle` → stash된 초대토큰 있으면 자동 참가, 없으면 홈.
- 엣지: 중복 레이스(확인 통과 후 submit 충돌) → 폼 유지하고 "이미 사용 중" 표시.

### 5.3 팀 선택 (홈 빈 상태, 자체 디자인)
일러스트/카피 + [모임 만들기](→ `/team-new`) · [초대 코드로 참가](→ 코드 입력 시트).

### 5.4 초대 코드로 참가 시트
코드 입력 → `joinByToken` → 응답별 처리:
- 성공: setCurrentTeam(team) → 홈, "참가 완료"
- `alreadyMember`: 해당 팀으로 전환, "이미 참가한 모임이에요"
- 만료/무효: "초대 코드가 만료됐거나 올바르지 않아요"

### 5.5 초대하기 (방장) — 멤버관리/모임설정에 진입
`generateInviteToken` → **초대 코드(텍스트) + QR + "X일 후 만료" + [복사] [공유]([share()]) [재생성]**.
QR은 딥링크 `intoss://pocketpay/join/<token>` 인코딩(순수 JS QR 매트릭스 → react-native-svg 셀 렌더).

### 5.6 모임 나가기 (멤버)
모임설정에 "모임 나가기"(확인 팝업) → `leaveTeam` → 팀 목록 갱신(0개면 팀 선택 빈 상태).
방장은 "위임 후 나갈 수 있어요" 안내(기존 계정탈퇴 경고와 동일 톤).

### 5.7 거래 추가 영수증 상단
`deal-new`에서 영수증 첨부 블록을 폼 맨 위로 이동.

### 5.8 프로필 핸들 쿨다운 정렬
`profile.tsx` 핸들 편집에 "N일 후 변경 가능"/쿨다운 에러 처리 통일.

## 6. 데이터 / API (프론트 신규)

- `api/team.ts`: `generateInviteToken(teamId)`, `joinByToken(token)`, `leaveTeam(teamId)`
- `lib/validation.ts`: handle 길이 3~20로 정렬(현재 함수 재사용)
- `lib/inviteToken.ts`(stash): storage에 pending token 저장/소비
- QR: 순수 JS 매트릭스 생성 + `components/common/QrCode.tsx`(svg Rect 렌더) — 네이티브 의존 없음
- teamStore: `joinTeam`, `leaveTeam` 액션(더미/실 분기), 신규 판별은 authStore에서

## 7. 예외처리 요약

- 핸들: 형식오류 / 빈값 / 중복 / (변경 시)쿨다운 / 레이스
- 초대코드: 만료·무효 / 이미 멤버 / 본인 모임
- 온보딩 이탈: handle 없으면 홈 진입 차단(게이트가 다시 온보딩)
- 딥링크: 미로그인·미온보딩 → stash 후 자동 소비, 만료 토큰이면 안내
- QR: 토큰 만료 시 재생성 버튼

## 8. 구현 순서 (각 1커밋)

1. 로그인 화면 리디자인
2. 온보딩 프로필 + 진입 게이트 (+ refreshUser 보장)
3. 팀 선택 빈 상태 + 초대코드 참가(joinByToken)
4. 초대 코드/QR 생성·표시·공유(방장) + QrCode 컴포넌트 + 딥링크 stash/자동참가
5. 모임 나가기(leaveTeam)
6. 거래추가 영수증 상단 이동 + 프로필 핸들 쿨다운 정렬 + 예외문구 정리

## 9. 범위 밖 (이번 아님)

- 인앱 QR 스캐너(카메라 직접 스캔) — 외부 카메라 스캔 + 텍스트 코드로 대체
- 실 푸시 발송 / 멀티슬라이드 앱 소개 튜토리얼
