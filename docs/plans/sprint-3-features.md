# Sprint 3 — 부가 기능 (통계 / 회비 / 멤버 초대 / 알림)

**목표**: 메인 PocketPay 1.1에 있는 부가 기능 중 토스 미니앱에서 가치 있는 것 이식.

**예상 기간**: 약 1주

**전제**: Sprint 2 통과 (핵심 동선 동작)

---

## 우선순위

토스 사용자에게 가장 가치 있는 순서로:

1. **알림** — 토스 자체 푸시 시스템 사용. 메인 repo의 push.service 로직 참고하되 토스 푸시 API로 재작성
2. **통계** — 월별 수입/지출, 카테고리별. TDS 차트 컴포넌트 사용
3. **회비** — 모임 회비 청구 + 납부 확인
4. **멤버 초대** — handle ID 기반 (메인 PocketPay v1.1과 동일 시스템)

## Task 1. 토스 푸시 알림 통합

- 토스 푸시 API 확인 (외부 FCM/APNs 불가)
- 백엔드 `notification.service.ts` 를 토스 푸시 호출 버전으로 분기
  - 메인 RN 앱 → expo-notifications (그대로)
  - 토스 미니앱 → 토스 푸시 API
- 권한 동의 모달 (필요 시점에만 — 회비 청구 보낼 때 등)

## Task 2. 통계 화면

- 월별 수입/지출 카드
- 카테고리별 분석 (TDS chart 컴포넌트)
- 전월 비교
- 백엔드 `/deals/stats` (또는 동등) 호출

## Task 3. 회비 관리

- 회비 청구 화면 (멤버 선택 + 금액 + 마감일)
- 납부 현황 (멤버별 납부/미납)
- 백엔드 `/fee/*` 호출

## Task 4. 멤버 초대

- handle ID 검색 → 사용자 표시 → 초대
- 메인 백엔드 `/users/search?handle=` 사용
- 초대 코드 입력 화면 (선택)

## Task 5. UI polish

- TDS 디자인 시스템 일관성 점검
- 토스앱 분위기에 맞는 마이크로 인터랙션
- 다크 모드 호환 (TDS가 기본 지원하면)

## Sprint 3 종료 시
- 친구 3~5명에게 베타 테스트
- 정책 위반 가능성 자체 점검 (외부 앱 유도 X / 결제는 토스페이/IAP만 / 등록명 일치 등)
- Sprint 4 진입 (출시 준비)
