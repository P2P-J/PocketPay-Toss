# Sprint 2 — 핵심 흐름 (인증 + 모임 + 거래 + OCR)

**목표**: 작은 모임의 핵심 동선(가입/로그인 → 모임 진입 → 거래 추가 → 영수증 OCR)이 토스 미니앱에서 동작.

**예상 기간**: 약 1주

**전제**: Sprint 1 통과 (첫 화면 + 백엔드 통신 OK)

---

## 결정 필요 사항 (시작 전)

1. **로그인 방식**: 토스 로그인 / 자체 이메일 가입 / 둘 다
   - 토스 로그인 → 사용자 친화적, 단 메인 PocketPay 사용자와 어떻게 연동할지 결정 필요
   - 자체 이메일 가입 → 메인 백엔드 그대로 사용. 단 토스 사용자 입장에서 마찰 큼
   - 추천: **토스 로그인 + 자체 사용자와 매핑** (Sprint 시작 시 커뮤니티/문서 확인)

2. **OCR 카메라**: 토스 SDK 카메라 vs HTML <input type="file">
   - Granite RN 환경에서 토스 카메라 SDK 사용 방식 확인 필요

## Task 1. 인증 화면

- 로그인 또는 가입 흐름
- 메인 백엔드 `/auth/*` 또는 새 토스 전용 라우트 호출
- 토스 사용자 식별자(있을 경우)와 자체 User 매핑 로직 설계

## Task 2. 모임 목록 화면

- 백엔드 `/teams` (또는 동등) 호출
- TDS 리스트 컴포넌트 사용
- 모임 선택 → 모임 상세

## Task 3. 모임 상세 + 거래 목록

- 백엔드 `/teams/:teamId/deals` 호출 (또는 동등)
- 카테고리별 색상/아이콘 (메인 repo의 `mobile/src/constants/categories.ts` 참조 후 옮김)
- 수입/지출 합계 표시

## Task 4. 거래 추가 화면

- 폼: 거래처 / 금액 / 날짜 / 카테고리 / 메모
- 백엔드 POST `/deals` (또는 동등)
- 성공 시 모임 상세 화면으로 복귀 + 목록 새로고침

## Task 5. 영수증 OCR 흐름

- 카메라 또는 사진 라이브러리에서 사진 선택
- 백엔드 POST `/ocr/analyze` 로 사진 업로드
- 백엔드 Clova OCR 결과 받아 거래 추가 폼에 자동 입력

**중요**: OCR 정확도는 백엔드 Clova가 결정 → 클라이언트는 사진 캡처 + 업로드만.

## Task 6. 통합 검증

- 신규 사용자 흐름: 가입 → 모임 만들기 → 거래 추가 → OCR
- 기존 사용자 흐름: 로그인 → 모임 진입 → 거래 추가

## Sprint 2 종료 시
- 친구/지인 1~2명에게 샌드박스 환경에서 사용해보게 함
- 피드백 받고 Sprint 3 우선순위 조정
